# Security Policy

## Supported Versions

We actively maintain and support the following versions of Muzify for security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

---

## Reporting a Vulnerability

We take the security of Muzify, our serverless APIs, proxies, and our users very seriously. If you find a security vulnerability, we appreciate it if you report it to us privately and responsibly instead of opening a public GitHub issue.

To report a vulnerability:

1. Send an email to **sagar@pixartual.studio** or **support@pixartual.studio** with details of the vulnerability.
2. Please include:
   - A description of the issue.
   - Steps to reproduce (proof of concept code, screenshots, or curl commands).
   - The potential impact of the vulnerability.
3. We will acknowledge your report within **48 hours** and provide a timeline for resolution.
4. We ask that you keep information about the vulnerability confidential until we have published a patch and announced it to protect all active deployments of the project.

Thank you for helping keep Muzify secure for everyone! 🔒
